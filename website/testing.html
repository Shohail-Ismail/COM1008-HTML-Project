contact.html

'contacts.html' is shown to have a syntax error stemming from using the 'placeholder' attribute with the <select> element, which is not valid as <select> offers a drop-down menu that cannot have a placeholder statement without the statement being one of the choosable options. Therefore, it is removed.

BEFORE:

AFTER:

Following through from the previous error, the first option of a required single-selection, drop-down <select> menu must be a placeholder option which is intended to be used to inform the user of how to use the menu. Although the error statement offers a 'size' attribute equal to the number of option elements be used, this is not the intended layout of the menu and would make it large and clunky, which is antithetical to the simplistic, modern design that is required. Therefore, a placeholder option label is added with no value for the 'value' attribute to ensure it will never be recorded by the server as a valid option.

AFTER:

After rectifying these errors, the page has been validated.

Browser testing

I tested 3 different browsers (Google Chrome, Mozilla Firefox emulator, and Microsoft Edge emulator) and found that all 3 can run the website as intended and all links direct to the page that they are required to. This is due to all 3 browsers adhering to standards set by the World Wide Web Consortium (W3C) and the Internet Engineering Task Force (IETF) for HTML5 and therefore render the pages identically.
Furthermore, by using the Google Chrome Remote Desktop tool, I was able to view the website on my phone, thereby getting a mobile view of the website.
